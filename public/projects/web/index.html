Personal Website

Web DesignWeb DevelopmentResearchIllustration

What do you do when nothing seems to match what you want?

I decided I might as well just learn how to make my portfolio website by myself, from scratch.

I already had a basic understanding of HTML and CSS, but going through multiple iterations of my personal website over the past year or two has been the most enlightening experience. I honestly wouldn't trade any of the bugs, sleepless nights, and next-day epiphanies for any drag-and-drop web design tool. (I had the luxury of being my own client and setting my own deadline, though. I'll own up to that!)

Throughout my journey so far I've gotten the hang of:

  • Eleventy, a static site generator
  • Basic Webpack tooling to help the JavaScript lighten up (it wasn't much but it was so gratifying)
  • Pug, an HTML pre-processor I absolutely love (that makes HTML so much more accessible for my eyes!)
  • SCSS mixins, partials, variables up the wazoo to help with palette flexibility site-wide
  • Netlify to deploy my website from GitHub
  • Netlify CMS, a headless CMS which I then learned to connect with Eleventy
  • Strapi as my most-recent and favored CMS thanks to its easy-on-the-eyes UI
  • Axios, GSAP, and other node packages
  • Figma components, properties, variants, and the like to really help visualize my indecisiveness
  • Mobile-first, responsive development and design that doesn't overly rely on media queries
  • and having the courage to admit sometimes a stray comma WILL get the better of me for an unacceptable amount of time and that we can just look at it again tomorrow
